Android Gradle构建流程

Posted on 2021-06-21 Edited on 2021-06-24 In gradle学习笔记

概述

Gradle是一个项目自动化构建开源工具，它通过组织执行一系列的Task来完成自动化的构建，减少构建项目时的复杂度。Android Studio也使用Gradle作为基础的构建工具。

Gradle自身是用Groovy和Java进行开发的，同时Gradle脚本使用基于Groovy的特定领域语言（DSL）来声明项目配置。
Gradle是一种DSL（Domain Specific Language），领域特定语言。说白了DSL就是一种行话，这个语言在Gradle中可以识别，在其它地方就用不了了。

学习Gradle，我们不必要精通Groovy语言，但还是需要了解一些Groovy的基本知识，比如def关键字、返回语句、字符串、集合、闭包等。下面，我们就来简单介绍一下。

Gradle项目结构分析

新建一个Android工程，项目的结构大致如下：

  • Project
    • .gradle
    • .idea
    • app
      • build
      • libs
      • src
      • .gitignore
      • build.gradle
      • …
    • gradle
    • submodule
    • .gitignore
    • build.gradle
    • gradle.properties
    • local.properties
    • settings.gradle
    • …
settings.gradle

每个项目整体只有一个settings.gradle文件，是用于配置整个项目结构的脚本，对应的是Gradle中的Settings对象（Settings.java）。其常用方法有：

  • include 依赖子工程
  • includeFlat 依赖平级工程
  • project 获取Project对象

一般子模块的引用 都是在settings.gradle中添加

rootProject/build.gradle

位于项目根目录下的build.gradle脚本，同样负责项目整体的配置，对应一个Project对象。常用方法有：

  • buildscript 配置构建脚本
  • repositories 配置仓库地址， 后面查找依赖的时候就按这个配置的顺序依次查找
  • dependencies 配置项目的依赖项
  • allprojects 所有项目的通用配置项
module/build.gradle

位于子模块的build.gradle脚本，负责这个模块的配置。也对应一个Project对象，类似根目录的build.gradle。

gradle.properties

gradle全局的统一配置，key-value配置，用于配置例如守护进程开关，aapt2开关，jvm参数和一些自定义的通用函数，需要上传到仓库

local.properties

和gradle.propertiies类似，主要是一些本地化的配置，像工程里需要源码依赖的模块，或者一些安全性较高的仓库账号密码
