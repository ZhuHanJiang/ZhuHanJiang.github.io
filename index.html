Kotlin 学习笔记（1）-- 内置类型

Posted on 2021-04-26 Edited on 2021-05-19 In Kotlin学习笔记

I. 基本类型

Kotlin Java
字节 Byte byte / Byte
整型 Int & Long int / Integer & long / Long
浮点型 Float & Double float / Float & double / Double
字符 Char char / Character
字符串 String String

1. val & var

val: 只读变量，等同于Java中 final变量，初始化后不可修改

var: 可读写变量， 和Java中的大部分变量一样

kotlin类型自动推导：在kotlin中声明变量可省略类型 : String ，kotlin会根据声明的值进行自动推导）

2. Long 类型

Java中数字结尾为 L / l 都可以编译，但在kotlin中 l 会编译错误

3. 数值类型转换

Kotlin中，隐式类型转换是不被允许的


必须使用显示转换：toLong(), toInt()等方式

4. 无符号类型（可兼容C语言）

无符号类型只能表示非负数，范围为对应有符号类型的二倍(bit位数一样，去掉负数占用的位置)
Int: $-2^{31}$ ~ $2^{31} - 1$
UInt: 0 ~ $2^{32} - 1$

有符号 无符号
字节 Byte UByte
短整型 Short UShort
整型 Int UInt
长整型 Long ULong
字符串 String String

无符号类型 不能声明为负数

5. String

kotlin中使用 字符串模板，不需要像Java一样用 “+” 拼接


kotlin 中 双等号==比较两个字符串的值， 三等号===比较两个字符串的引用（即Kotlin中===
和JAVA中的==作用一致）

==只能用于比较同一类的两个实例，不同类型之间比较会报错

6. Raw String

打印出来的格式会和直接输入进去的格式一模一样，所有的空格 & 变量都会包含
trimIndent()用于把前面的公共空格消掉

7. 位运算

Kotlin对于位运算，没有特殊字符，只可用中缀方式调用具名函数


完整的位运算列表（只用于 Int 与 Long）:

  • shl(bits) – 有符号左移
  • shr(bits) – 有符号右移
  • ushr(bits) – 无符号右移
  • and(bits) – 位与
  • or(bits) – 位或
  • xor(bits) – 位异或
  • inv() – 位非

II. 数组

Kotlin中数组用Array类表示，包含：get与set函数（运算符重载为[]）,size属性

Kotlin Java
整型 IntArray int[]
整型装箱 Array Integer[]
字符 CharArray char[]
字符装箱 Array Character[]
字符串 Array String[]
自定义类 Array

基本数据类型都有对应的xxArray,(Long\Float\Double\Byte)

1. 创建数组

库函数arrayOf(vararg 变长参数)或arrayOfNulls(size)


构造函数Array(size, init: (int) -> int)

2. 数组读写

运算符重载：[]代表调用成员函数get() & set(value)


Kotlin 中数组是不型变的（invariant）。这意味着 Kotlin 不让我们把 Array<String> 赋值给 Array<Any>，以防止可能的运行时失败

3. 查找特定element

any{ (element) -> Boolean }: 存在任意一个元素返回true -> true, 所有元素都返回false -> false
any(): Returns true if this array has at least one element

III. 区间

区间是Java中并没有的数据结构，在范围内遍历的时候十分方便

1.创建

区间的类型为：

  • 离散区间：IntRange/ UIntRange, CharRange, LongRange/ ULongRange
  • 连续区间：ClosedFloatingPointRange (不可数)

2.打印

连续区间不可数，也不可使用joinToString()直接打印所有数值

3.迭代（仅离散区间可用）

和Java方式差不多

4.查找element

  • 离散区间：可以使用any{}，和数组的方式一样。也可以使用if (element in range)
  • 连续区间：由于不可数，any{}方法不适用。可使用if (element in range)的方式

5.应用

在指定区间内遍历的时候，可使用区间直接给出范围和步长。


在kotlin中还有很多其他的用途，后续再慢慢发现

反射是指允许程序在运行时访问程序结构的一类特性
这些结构包括：类、接口、方法、属性等语法特性

2. 用途

  • 拿到类型的：属性、方法、内部类
  • 调用指定名称/签名的方法，或访问指定名称的属性（只要有类名 —> 能拿到类 —> 拿到类的属性、方法等等）
  • 通过签名信息获取泛型实参的具体类型（编译之后，泛型参数是要被擦除掉的，字节码中只有原始类型）
  • 访问运行时注解及其信息完成注入或者配置操作

3. 反射常用数据结构

数据结构 Java中对应结构 说明
KType Type 描述未擦除的类型或泛型参数，eg：Map。可通过typeOf或以下类型获取对应的父类、属性、函数参数等
KClass Class 描述对象的实际类型，不包含泛型参数，eg：Map；可通过对象、类型名直接获得
KProperty Field 描述属性，可通过属性引用、属性所在类的KClass获取
KFunction Method 描述函数，可通过函数引用、函数所在类的KClass获取

KType & KClass:

KType描述的就是我们直接使用的类型(Map)；KClass描述的是真实的类型(Map)，泛型擦除后的实际类型

KProperty & Field：

KProperty 比 Field 更强大。Kotlin中 property = field + setter + getter

KFunction & Method：

KFunction 既可以是顶级函数、也可以是类的方法函数。但Java中没有顶级函数概念，Method仅指类的方法

4. 使用反射

Java反射：

  • Pros：在Kotlin中可以直接使用、并且无需引入额外依赖
  • Cons：无法使用kotlin的语法特性，需对kotlin生成的字节码足够了解

    Kotlin反射：

  • Pros: 支持Kotlin中的几乎所有特性，API也更友好
  • Cons: 需要添加 kotlin反射库 的引用，首次调用慢

kotlin的反射信息是写在@Metadata里面的，所以首次调用有一个获取注解、反序列化信息的过程
